# Contributing to Horizon

Thank you for considering contributing to Horizon! We appreciate your interest and welcome your contributions. By contributing, you help improve the project and make it more valuable to the community.

## How to Contribute

### Bug Reports

If you encounter a bug or unexpected behavior, please open an issue on the GitHub repository. When submitting a bug report, provide as much detail as possible, including steps to reproduce the issue, your environment details, and any error messages you received.

### Feature Requests

If you have an idea for a new feature or enhancement, feel free to open an issue to discuss it. Clearly describe the proposed feature and its use case so that the community can provide valuable feedback.

### Code Contributions

We welcome code contributions in the form of pull requests (PRs). Before submitting a PR, please follow these steps:

1. **Fork the Repository:** Fork the repository to your GitHub account.

2. **Clone the Repository:** Clone the forked repository to your local machine.

    ```bash
    git clone https://github.com/Cameleon00722/horizon.git
    ```

3. **Create a Branch:** Create a new branch for your changes.

    ```bash
    git checkout -b feature/your-feature-name
    ```

4. **Make Changes:** Implement your changes. Ensure your code follows the project's coding standards and practices.

5. **Run Tests:** If applicable, run tests to ensure your changes do not introduce regressions.

6. **Commit Changes:** Commit your changes with descriptive commit messages.

    ```bash
    git commit -m "Add your commit message here"
    ```

7. **Push Changes:** Push your branch to your forked repository.

    ```bash
    git push origin feature/your-feature-name
    ```

8. **Create a Pull Request:** Open a pull request against the main repository's `main` or `master` branch. Clearly describe the purpose of your PR and provide any additional information.

### Code Review

All code changes will go through a review process. Reviewers will provide feedback, and you may need to address any requested changes before your contribution is accepted.

## Code of Conduct

Please review and adhere to our [Code of Conduct](CODE_OF_CONDUCT.md) when participating in this project. We aim to foster a positive and inclusive community.
